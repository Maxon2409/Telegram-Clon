# Распространение через поиск (без Play Market и App Store)

Чтобы люди находили приложение в Google, Яндексе и могли скачать или открыть на телефоне — нужен **свой сайт** с приложением.

---

## Вариант 1: Веб-приложение (самый простой)

Твоё приложение уже умеет работать в браузере (`npx expo start --web`). Его можно выложить в интернет.

### Что получается
- Человек ищет в Google/Яндексе → находит твой сайт → открывает приложение **прямо в браузере** на телефоне.
- Ничего качать не нужно: зашёл по ссылке и пользуешься.
- В браузере на телефоне можно нажать «Добавить на главный экран» — иконка будет как у приложения (PWA).

### Как выложить (бесплатно)

1. **Собрать веб-версию:**
   ```bash
   npx expo export --platform web
   ```
   Появится папка `dist/` с готовым сайтом.

2. **Загрузить на хостинг** (один из вариантов):
   - **Vercel** — [vercel.com](https://vercel.com), привязка к GitHub, автодеплой.
   - **Netlify** — [netlify.com](https://netlify.com), загрузка папки `dist` или из Git.
   - **GitHub Pages** — бесплатно, если проект в GitHub.

3. **Купить домен** (по желанию), например `telegram-clon.ru`, и привязать к этому хостингу.

4. **Настроить поиск (SEO):**
   - На странице есть заголовок и описание (в `app.json` или в веб-шаблоне).
   - После публикации добавь сайт в [Яндекс.Вебмастер](https://webmaster.yandex.ru) и [Google Search Console](https://search.google.com/search-console) — тогда сайт начнут индексировать, и люди будут находить его в поиске.

Итог: **поиск (Яндекс/Google) → переход на твой сайт → приложение открывается в браузере на телефоне.**

## Вариант 2: Сайт + кнопка «Скачать на Android»

Если хочешь, чтобы люди именно **скачивали файл на телефон**:

1. Сделай всё из варианта 1 (сайт в поиске).
2. Собери APK для Android:
   ```bash
   eas build --platform android --profile preview
   ```
   Или локально: `npx expo run:android` и взять APK из сборки.
3. Залей APK на свой сайт (или на файлообменник с постоянной ссылкой).
4. На главной странице сайта добавь кнопку **«Скачать для Android»** с ссылкой на этот APK.

Тогда:
- В поиске (Google/Яндекс) люди находят твой сайт.
- На телефоне нажимают «Скачать» → скачивается APK.
- Устанавливают (Android попросит разрешить установку из неизвестных источников).

**Важно:** для iPhone установка приложений не с App Store почти невозможна для обычных пользователей. Для них остаётся только открыть твой сайт в браузере (вариант 1) или «Добавить на главный экран».

## Вариант 3: Альтернативные магазины (только Android)

Есть магазины вне Google Play, например:
- **RuStore** (Россия)
- **APKPure**, **F-Droid**

Там тоже можно опубликовать APK. Но **поиск в Google/Яндекс** ведёт обычно на сайт, а не сразу в эти магазины. Поэтому логика такая: сайт (варианты 1–2) — чтобы люди находили через поиск; альтернативные магазины — дополнительный канал распространения.

---

## Кратко

| Цель | Что делать |
|------|------------|
| Чтобы **находили в поиске** (Google, Яндекс) | Выложить веб-версию на хостинг (Vercel/Netlify/GitHub Pages), настроить SEO и Вебмастеры. |
| Чтобы **не качали, а сразу открывали** на телефоне | Пользоваться веб-версией по ссылке или «Добавить на главный экран» (PWA). |
| Чтобы **скачивали APK на Android** | На том же сайте сделать кнопку «Скачать для Android» со ссылкой на APK. |
| Для **iPhone** без App Store | Только веб-версия по ссылке или иконка на главном экране. |

Главное: **любой поисковик (Яндекс, Google, Хром и т.д.) находит только сайты.** Поэтому сначала делаешь сайт с приложением (или с кнопкой скачивания APK), потом люди смогут найти его через поиск и открыть/скачать на телефон.
