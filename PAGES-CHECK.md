# Если на сайте всё ещё виден README

Сделай по шагам и проверь каждый пункт.

---

## 1. Workflow должен быть зелёным

1. Открой: https://github.com/Maxon2409/Telegram-Clon/actions  
2. Открой последний запуск **Deploy to GitHub Pages**.  
3. Должна быть **зелёная галочка**. Если красный крестик — нажми на упавший шаг и пришли текст ошибки.

Если упал шаг **Build web** — сборка не прошла, ветка `gh-pages` не обновляется.  
Если упал **List build output** — в проекте нет папки `dist` после сборки.

---

## 2. Должна быть ветка gh-pages

1. На главной репозитория (https://github.com/Maxon2409/Telegram-Clon) нажми на выпадающий список с веткой (сейчас там **main**).  
2. В списке должна быть ветка **gh-pages**. Если её нет — workflow либо не запускался, либо упал до шага Deploy.  
3. Выбери **gh-pages** и посмотри файлы. Должны быть **index.html** и папки вроде **_expo**, **assets**. Если там только README или пусто — деплой не сработал.

---

## 3. В Pages должен быть выбран источник «ветка» и ветка gh-pages

1. Открой **Settings** репозитория → слева **Pages**.  
2. В блоке **Build and deployment**:  
   - **Source:** должно быть **Deploy from a branch** (не GitHub Actions).  
   - **Branch:** обязательно выбери **gh-pages** (не main).  
   - **Folder:** оставь **/ (root)**.  
3. Нажми **Save**.

Если здесь выбрана ветка **main**, сайт будет показывать README, а не приложение.

---

## 4. Открывай именно ссылку сайта и без кэша

- Правильная ссылка: **https://maxon2409.github.io/Telegram-Clon/**  
- Не репозиторий: не https://github.com/Maxon2409/Telegram-Clon  

Открой ссылку в режиме инкогнито или с принудительным обновлением (Ctrl+F5), чтобы не показывался старый кэш.

---

## 5. После смены ветки подожди 1–2 минуты

После смены ветки на **gh-pages** в Settings → Pages подожди 1–2 минуты и снова открой https://maxon2409.github.io/Telegram-Clon/
